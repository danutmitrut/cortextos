# 09 — Dashboard Tasks Page

Duration target: 12-15 minutes

## Learning Outcomes

- Understand task statuses and ownership.
- Review deliverables before accepting work.
- Spot blocked or invisible work.

## Script

Explain that tasks are cortextOS's work ledger. If work matters, it should be a task. Show statuses: pending, in progress, blocked, completed, and any review state the dashboard exposes.

Open a task and explain what a reviewer needs: clear title, owner, description, current status, result summary, and attached file deliverables. Emphasize that notes are not deliverables; saved output files are the reviewable artifacts.

Demonstrate a blocked task. Explain human blockers, agent blockers, and approval blockers. The goal is not to hide blockers; it is to make them visible so the system can recover.

## Screen Plan

| Time | Show On Screen |
|---|---|
| 0:00 | Tasks page filters and columns. |
| 2:00 | In-progress task. |
| 4:30 | Completed task with deliverable attachment. |
| 7:00 | Blocked task with reason. |
| 10:00 | Task activity or related output. |

## Learner Checkpoint

Create or find one task and verify whether it has enough information for another human to review it.
