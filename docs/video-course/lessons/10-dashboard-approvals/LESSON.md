# 10 — Dashboard Approvals Page

Duration target: 10-12 minutes

## Learning Outcomes

- Understand when agents must ask for approval.
- Review approval context before deciding.
- Distinguish permission blockers from capability blockers.

## Script

Introduce approvals as the safety surface for actions that affect the outside world: sending emails, posting publicly, deploying, deleting data, spending money, or making irreversible changes.

Show a pending approval. Read the title, category, context, proposed action, and linked task. Explain that a good approval gives enough context to say yes or no without asking the agent to repeat itself.

Then reject or approve a demo approval and show the downstream effect. Approved work resumes; rejected work should be closed or revised.

## Learner Checkpoint

List three actions your agents should never take without approval.
