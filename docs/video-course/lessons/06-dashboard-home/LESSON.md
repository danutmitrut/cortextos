# 06 — Dashboard Home

Duration target: 8-10 minutes

## Learning Outcomes

- Read the dashboard home page as the operator overview.
- Know what to inspect first when deciding whether the system is healthy.

## Script

Frame the home page as the "morning glance" for cortextOS. It should answer: are agents online, is work moving, are there approvals or blockers, and did anything important happen since the last check?

Walk top to bottom. Explain that home is not where every detail lives; it is where you decide where to click next. If an agent is stale, go to Agents. If work is blocked, go to Tasks. If a risky action is waiting, go to Approvals. If you need audit context, go to Activity.

Close with an operator habit: every session starts with home, then one focused page based on the signal.

## Screen Plan

| Time | Show On Screen |
|---|---|
| 0:00 | Dashboard home page full screen. |
| 1:00 | Agent status cards or summary section. |
| 2:30 | Task/approval summaries. |
| 4:30 | Recent activity section. |
| 6:30 | Navigate from home to the most relevant detail page. |

## Learner Checkpoint

Name the first page you would open if the home page shows a stale agent, a blocked task, or a pending approval.
