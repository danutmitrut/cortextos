# 31 — Memory and Knowledge Workflows

Duration target: 15-18 minutes

## Learning Outcomes

- Understand daily memory, long-term memory, and KB.
- Write memory that helps future sessions resume.
- Decide what belongs in structured data versus narrative memory.

## Script

Explain the three layers. Daily memory is the current session journal. Long-term memory is synthesized durable knowledge. The knowledge base is semantic retrieval across documents and prior outputs.

Show a daily memory entry, `MEMORY.md`, and a KB query. Explain when to use each. A decision, preference, or handoff goes to memory. A research report or reusable doc goes to KB. A CRM contact belongs in structured CRM files or tools, with relevant summary memory only where appropriate.

Close by demonstrating a restart-resume scenario: a good memory checkpoint lets an agent continue without rereading the whole repo.
