# 15 — Dashboard Skills Page

Duration target: 10-12 minutes

## Learning Outcomes

- Explain what skills are.
- Find available skills.
- Decide when a repeated workflow deserves a skill.

## Script

Define a skill as reusable operating instructions for a class of work. Skills are how agents become better at repeatable workflows without re-learning them from scratch every time.

Show the skills page and one `SKILL.md` file. Explain progressive disclosure: a skill should tell the agent what to do, when to use it, and where supporting scripts/templates live.

Close with examples: approvals skill, tasks skill, knowledge-base skill, and a domain skill like CRM management.

## Screen Plan

| Time | Show On Screen |
|---|---|
| 0:00 | Skills page. |
| 2:00 | Search/filter for a known skill. |
| 4:00 | Open a skill file in editor. |
| 6:30 | Show how a task invokes a skill. |
| 9:00 | Mention skill creation threshold: repeated workflow or complex task. |

## Learner Checkpoint

Name one workflow in your setup that should become a reusable skill.
