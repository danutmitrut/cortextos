# 12 — Dashboard Activity Page

Duration target: 8-10 minutes

## Learning Outcomes

- Read the activity feed as an audit trail.
- Know which events are meaningful for operations.
- Use events to reconstruct what happened.

## Script

Frame Activity as the timeline of the system. Tasks, sessions, outputs, approvals, cron completions, errors, and major decisions should show up here.

Walk through normal events first, then error events. Explain severity and metadata. A good activity trail lets a reviewer reconstruct a session without asking the agent to summarize from memory.

Close by connecting this page to trust: agents are more useful when their actions are visible and inspectable.

## Screen Plan

| Time | Show On Screen |
|---|---|
| 0:00 | Activity feed. |
| 1:30 | Session start/task created events. |
| 3:00 | Output created/task completed events. |
| 5:00 | Approval or blocked event. |
| 7:00 | Error or warning event example. |

## Learner Checkpoint

Find the latest completed task event and trace it back to the task page.
