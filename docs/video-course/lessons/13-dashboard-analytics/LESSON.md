# 13 — Dashboard Analytics Page

Duration target: 8-10 minutes

## Learning Outcomes

- Interpret analytics without overfitting.
- Use metrics to improve operations.
- Distinguish signal from vanity data.

## Script

Introduce analytics as the pattern view: throughput, responsiveness, agent activity, task completion, experiment results, or cost/performance where available.

Explain that analytics should drive decisions. If an agent has many in-progress tasks, improve task hygiene. If approvals pile up, improve approval context or review cadence. If outputs are low quality, improve templates, skills, or task prompts.

Close with a warning: metrics are a conversation starter, not an excuse to ignore the actual task artifacts.

## Screen Plan

| Time | Show On Screen |
|---|---|
| 0:00 | Analytics page overview. |
| 2:00 | Task/agent trend chart. |
| 4:00 | Approval or throughput metric. |
| 6:00 | Drill from metric to task/activity evidence. |

## Learner Checkpoint

Choose one metric and name one operational change it suggests.
