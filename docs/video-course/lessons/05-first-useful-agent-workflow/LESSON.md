# 05 — First Useful Agent Workflow

Duration target: 15-18 minutes

## Learning Outcomes

- Turn a vague request into a tracked cortextOS task.
- Attach a file deliverable.
- Close the loop with dashboard-visible status.

## Script

Start with a practical request: "Research three starter automations I should build this week and save the plan." Ask the agent through Telegram. Then show what should happen in cortextOS: a task is created, marked in progress, work is done, an output file is saved with a human-readable label, and the task is completed.

Explain why this matters. cortextOS is not just about generating text; it is about creating a work trail that survives restarts and lets humans review what actually shipped.

Show the output file and task page side by side. Point out the difference between notes and deliverables: notes provide context, but deliverables are attached artifacts.

Close by having the learner repeat the loop with a tiny request, such as a one-page "first week agent plan."

## Screen Plan

| Time | Show On Screen |
|---|---|
| 0:00 | Telegram request to the agent. |
| 2:00 | Dashboard task appears as in progress. |
| 5:00 | Terminal or editor with generated output file. |
| 8:00 | Bus command or dashboard action attaching saved output. |
| 11:00 | Completed task with deliverable label visible. |
| 14:00 | Activity feed showing task lifecycle events. |

## Learner Checkpoint

Complete one small task that has at least one saved output attached.
